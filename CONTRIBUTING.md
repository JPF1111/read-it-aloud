# Contributing to Read It Aloud

Thanks for the interest. Here's how participation works on this project.

## Code contributions

**This repository does not accept code pull requests.**

Source code development happens in a separate private repository so that
the licensed code stays controlled. Read It Aloud is distributed under
the [PolyForm Noncommercial License](./LICENSE) — free for personal,
educational, and noncommercial use, with commercial use reserved.

If you want to contribute code, the best paths are:

- **Open an issue** describing what you'd like to change and why. If it
  fits the roadmap, the maintainer will implement it.
- **Sponsor the project** so the maintainer has the runtime to ship
  what you want faster. See [the support links](./README.md#support-the-project).
- **Reach out about a commercial license** if you want to build a derivative
  product. See [LICENSE](./LICENSE).

## Bug reports

Bug reports are welcome and high-value. Open an issue at
<https://github.com/JPF1111/read-it-aloud/issues> and include:

- Browser and version (Chrome 138, Firefox 130, Safari 17, etc.)
- Extension version (visible at the bottom of the popup)
- Operating system
- Steps to reproduce
- What you expected
- What actually happened
- Any console errors (open DevTools → Console)

For security issues, see [SECURITY.md](./SECURITY.md) — do **not** open
a public issue for vulnerabilities.

## Feature requests

Feature requests are also welcome. Open an issue and explain:

- What problem you're trying to solve
- How you'd use the feature
- Any constraints (accessibility need, language, browser, etc.)

The roadmap is shaped heavily by what makes the tool more useful for
people with ADHD, dyslexia, and other reading-accessibility needs. Asks
that align with that mission tend to ship faster.

## Translations and accessibility

Help translating the UI into other languages, or testing with assistive
technology (screen readers, magnifiers, voice control), is genuinely
appreciated. Open an issue tagged `i18n` or `a11y` to start the conversation.

## Communication standards

Be kind. Be specific. Assume good faith. This project exists to help
people whose brains and eyes don't work like the default web assumes —
that audience deserves a project space that mirrors that care.

The maintainer reserves the right to close issues that are abusive,
demanding, off-topic, or that effectively expect free commercial labor.

## Saying thanks

If the tool helped you, a [coffee](https://buymeacoffee.com/ReadItAloud)
or a star on this repository is more than enough. There is no expectation
of payment — the extension is and will remain free.
